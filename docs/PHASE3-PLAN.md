## 🎯 Phase 3A: 复杂 SoC 集成

### 目标
构建一个完整的 RISC-V SoC 系统，集成：
- RISC-V 核心 (简化版)
- SRAM 控制器
- UART 控制器
- GPIO 控制器
- 中断控制器

### 架构框图
```
┌─────────────────────────────────────────────────────────┐
│                      RISC-V SoC                         │
│  ┌──────────┐     ┌──────────┐     ┌──────────┐        │
│  │ RISC-V   │────▶│ AXI4     │────▶│ SRAM     │        │
│  │ Core     │     │ Interconnect│  │ Controller│        │
│  └──────────┘     └──────────┘     └──────────┘        │
│       │                │                │               │
│       ▼                ▼                ▼               │
│  ┌──────────┐     ┌──────────┐     ┌──────────┐        │
│  │ UART     │     │ GPIO     │     │ Timer    │        │
│  │ Controller│    │ Controller│    │          │        │
│  └──────────┘     └──────────┘     └──────────┘        │
└─────────────────────────────────────────────────────────┘
```

### 任务分解

| Task | 描述 | 优先级 | 工时 |
|------|------|--------|------|
| T301 | RISC-V RV32I 核心简化版 | P0 | 2 天 |
| T302 | AXI4-Lite Interconnect | P0 | 1 天 |
| T303 | SRAM 控制器 (64KB) | P0 | 1 天 |
| T304 | 中断控制器 (PLIC 简化) | P1 | 1 天 |
| T305 | 系统集成与验证 | P0 | 1 天 |

### 验收标准
- ✅ RISC-V 核心运行简单循环程序

## 🎯 Phase 3B: AXI4 总线系统

### 目标

- AXI4-Lite 主设备

- 地址解码

│  │ Master 0 │────▶│          │────▶│ Slave 0  │        │
│  │ (CPU)    │     │  AXI4    │     │ (SRAM)   │        │
│  └──────────┘     │  Matrix  │     └──────────┘        │
│  ┌──────────┐     │          │     ┌──────────┐        │

## 🔧 技术挑战与风险

### 技术挑战

| 挑战 | 风险等级 | 缓解措施 |
|------|----------|---------|
| RISC-V 核心时序收敛 | 🟡 中 | 简化流水线，降低频率目标 |
| AXI4 协议复杂性 | 🟡 中 | 使用 AXI4-Lite 简化版 |
| 视频处理带宽需求 | 🟠 高 | 使用双缓冲，优化访问模式 |

### 依赖关系

```
Phase 3A (SoC) ──────▶ 需要 Phase 1 UART/GPIO/Timer
Phase 3B (AXI4) ──────▶ 需要 Phase 3A 基础架构
Phase 3C (Video) ──────▶ 需要 Phase 1 VGA + Phase 3B AXI4
```
