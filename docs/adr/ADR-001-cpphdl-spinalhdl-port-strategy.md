# ADR-001: CppHDL SpinalHDL 移植策略

**状态**: ✅ 已采纳  
**日期**: 2026-03-30  
**决策人**: 老板 + DevMate  
**评审记录**: om_x100b53a93bdd14bcc36db4723a80887

---

## 背景

为将 SpinalHDL 经典示例移植到 CppHDL，需要明确技术策略和优先级，确保开发效率与代码质量的平衡。

---

## 决策

### D1: SpinalHDL 兼容层策略

**决策**: **B) 语义兼容，C++ 风格**

**理由**:
- 保持 CppHDL 的 C++ 生态优势
- 降低 C++ 开发者学习成本
- 复用 SpinalHDL 设计思想，但不盲目模仿 API

**影响**:
- Stream 管道操作符命名为 `stream_m2s_pipe()` 而非 `m2sPipe()` 成员函数
- 同时提供成员函数别名 `stream.m2sPipe()` 以方便 SpinalHDL 用户
- 文档中同时展示两种风格

---

### D2: CDC 支持优先级

**决策**: **B) 分阶段实现**

**理由**:
- Phase 0/1 的基础示例（Counter、FIFO、UART）均为单时钟
- CDC 实现复杂，需要充分测试验证
- 避免 CDC 复杂性阻塞基础功能开发

**阶段规划**:
- **Phase 0** (本周): 单时钟基础功能
- **Phase 1** (2 周): 状态机、断言、位宽适配器
- **Phase 2** (3 周): CDC 支持（双时钟 FIFO、格雷码、同步器）

**影响**:
- VGA/RISC-V 等复杂示例延后
- 先完成 80% 基础示例移植

---

### D3: TDD 严格程度

**决策**: **B) 混合模式**

**理由**:
- 核心功能（Stream、CDC、内存）需要严格测试保证质量
- 示例代码以演示为主，可后补测试
- 平衡开发速度与质量

**执行规则**:
- **核心库** (`include/chlib/`): 严格 TDD，先测试后实现
- **示例** (`examples/`): 先实现，后补基础测试
- **工具链** (代码生成、仿真): 关键路径 TDD，辅助功能可后补
