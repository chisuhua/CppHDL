## 1. 产品概述

### 1.1 产品名称

**CppHDL** — C++ Hardware Description Library

### 1.2 产品定位

CppHDL 是一个基于现代 C++（C++20）的高层次硬件描述库，灵感来源于 Chisel 和 SpinalHDL。它允许数字电路设计师使用 C++ 的表达能力来描述硬件、进行仿真验证，并生成可综合的 Verilog 代码。

CppHDL 的核心价值主张是：**让熟悉 C++ 的工程师能够用自己最擅长的语言进行硬件设计，无需学习 Scala（Chisel）或其他 DSL。**

### 1.3 产品愿景

成为 C++ 生态中最成熟、最易用、最高性能的硬件描述工具链，填补 C++ 开发者进入硬件设计领域的工具空白。

## 2. 目标用户

### 2.1 主要用户画像

| 画像 | 描述 | 核心需求 |
|------|------|---------|
| **C++ 背景的 FPGA/ASIC 设计师** | 有 C++ 开发经验，需要设计数字电路，但不熟悉 Scala/Verilog | 用 C++ 描述硬件，仿真验证，生成 Verilog |
| **嵌入式软件工程师** | 需要设计简单的外围逻辑（GPIO、UART、SPI），希望快速原型 | 简单的 API、快速仿真、可综合输出 |
| **数字电路教学者/学生** | 需要教学工具来演示数字电路概念 | 清晰的语法、即时的仿真反馈、可视化的波形 |
| **RISC-V 架构师** | 需要快速迭代 CPU 设计，验证架构决策 | 高性能仿真、完整的 AXI 总线支持、Cache 模型 |

### 2.2 非目标用户

- **纯 Verilog/VHDL 用户**（无意迁移到高层次工具）
- **SystemVerilog/UVM 验证工程师**（CppHDL 不提供完整的验证方法学）
- **模拟/混合信号设计师**（CppHDL 仅支持数字电路）

## 4. 技术架构概要

### 4.1 架构风格

CppHDL 采用**组件化 + 流水线处理**的架构：

```
C++ DSL 代码
    ↓
AST DAG 构建（lnodeimpl 图）
    ↓
拓扑排序（context::get_eval_list）
    ↓
    ├──→ 仿真引擎（解释器 / JIT）
    ├──→ Verilog 代码生成
    └──→ DAG 可视化
```

### 4.2 核心组件

| 组件 | 职责 | 关键技术 |
|------|------|---------|
| **context** | 节点存储、拓扑排序、全局状态 | thread_local 上下文 |
| **lnodeimpl** | AST 节点基类、DAG 图构建 | C++ 多态、智能指针 |
| **Simulator** | 仿真执行（解释器 + JIT 双模式） | LLVM ORC JIT |
| **chlib** | 标准库组件 | C++20 模板、概念约束 |
| **CodeGen** | Verilog/DAG 代码生成 | AST 遍历、模板展开 |

### 4.3 关键技术选型

| 决策点 | 选择 | 理由 | ADR 参考 |
|--------|------|------|---------|
| 语言版本 | C++20 | 概念约束、consteval、三路比较 | — |
| 构建系统 | CMake 3.14+ | 跨平台、依赖管理 | — |
| 测试框架 | Catch2 v3.7 | 单头 amalgamation、BDD 风格 | — |
| JIT 编译器 | LLVM ORC JIT | 运行时编译、高性能 | ADR-003（✅ 已创建） |
| 内存管理 | 混合（裸指针 + 智能指针） | 性能与安全的权衡 | ADR-007（待创建） |
| SpinalHDL 兼容 | 语义兼容，C++ 风格 | 平衡复用性和 C++ 生态 | ADR-001 |

## 5. 非目标（明确不做什么）

CppHDL **不会**涉及以下领域：

1. **模拟/混合信号仿真** — 专注于数字电路
2. **完整的 UVM 验证方法学** — 提供基础断言，但不构建完整验证框架
3. **物理综合/布局布线** — 输出 Verilog 后交由 EDA 工具链处理
4. **图形化设计界面** — 专注于代码驱动的 DSL
5. **与其他 HDL 语言的互操作** — 不直接支持 Verilog/VHDL 混合仿真
6. **商业 EDA 工具的替代** — CppHDL 是前端 DSL，不是完整的 EDA 套件

## 6. 技术约束

### 6.1 硬性约束

| 约束 | 说明 |
|------|------|
| **C++20 编译器** | GCC 13+ / Clang 16+ / MSVC 19.35+ |
| **LLVM 17-19** | JIT 功能的依赖（可选） |
| **64 位平台** | macOS / Linux / Windows |
| **无外部运行时** | 库本身无运行时依赖（LLVM 仅在编译时需要） |

### 6.2 软性约束

| 约束 | 说明 |
|------|------|
| **头文件为主** | 核心库尽量使用 header-only，减少编译依赖 |
| **单头入口** | `#include "ch.hpp"` 即可使用全部功能 |
| **无全局状态污染** | 使用 thread_local 隔离，支持多 context |

## 附录 A：术语表

| 术语 | 定义 |
|------|------|
| **AST** | Abstract Syntax Tree，抽象语法树 |
| **DAG** | Directed Acyclic Graph，有向无环图 |
| **JIT** | Just-In-Time，即时编译 |
| **LLVM ORC** | LLVM 的 On-Request Compilation JIT 框架 |
| **Bundle** | CppHDL 中的结构体类型，类似 SpinalHDL 的 Bundle |
| **Stream** | 基于 valid/ready 握手协议的数据流 |
| **CDC** | Clock Domain Crossing，跨时钟域 |
| **SSO** | Small Buffer Optimization，小对象优化 |
| **VCD** | Value Change Dump，标准波形文件格式 |

## 附录 B：与竞品的对比

| 特性 | CppHDL | Chisel | SpinalHDL |
|------|--------|--------|-----------|
| 宿主语言 | C++ | Scala | Scala |
| 学习曲线 | C++ 开发者友好 | 需学 Scala | 需学 Scala |
| 仿真速度 | 解释器 + JIT | 仅解释器（慢） | 编译为 Verilog 后仿真 |
| 代码生成 | Verilog | Verilog | Verilog/VHDL |
| 类型系统 | C++ 模板（编译期检查） | Scala 类型 + 运行期检查 | Scala 类型 + 运行期检查 |
| JIT 编译 | ✅（LLVM ORC） | ❌ | ❌ |
| 社区规模 | 小（个人项目） | 大（Berkeley 支持） | 中 |
| 工业采用 | 实验性 | 高（SiFive 等） | 中 |
