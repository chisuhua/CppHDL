## Issue #1: sync_fifo 写使能逻辑缺陷

**严重级别**: 🔴 高

**问题描述**:
`sync_fifo` 函数中，写使能条件 `write_enable = wren && !is_full(count)` 存在问题：
- `count` 是寄存器，在组合逻辑中是**上一周期的值**
- 导致写入判断基于过时的计数值
- 实际表现：数据无法正确写入 FIFO

**影响范围**:
- 所有使用 `sync_fifo` 的设计
- FIFO 相关示例无法正常工作

**根本原因**:
组合逻辑中使用寄存器当前值（旧值）进行条件判断，而非下一周期值。

**修复方案**:
1. 将 `write_enable` 逻辑改为时序逻辑
2. 或者在 `describe()` 中明确指定写使能时序

**临时规避**:
使用 `ch_mem` 直接实现 FIFO，绕过 `sync_fifo` 函数。

## Issue #2: ch_mem 读端口 API 复杂

**严重级别**: 🟡 中

**问题描述**:
`ch_mem::sread()` 返回类型复杂，需要 `read_data <<= memory.sread(...)` 语法，不够直观。

**影响范围**:
- FIFO 实现
- 所有使用内存读端口的模块

**建议改进**:
简化内存读 API，如 `memory.read(addr)` 直接返回数据。
